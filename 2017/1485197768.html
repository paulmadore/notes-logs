Staying On Task

Most of my life, I didn’t work for myself, or set my own schedule. Most of my life, I did crap that I didn’t want to do for a living. When I finally got the chance to work for myself, I jumped at it. All those years I had spent cramming hackathons into my weekends – I would be great. And at first, I was. But then, you develop habits. You get on Facebook too long, etc. There are a lot of things you can do to alleviate the distraction portion of life, but the thing I’ve done to ensure that I get back on task fast is to create a little script that pops open my new “primary_work” folder. That’s all it does. That’s all it has to do. The whole thing is right here:

#!/usr/bin/env python
import os
os.system('thunar ~/Desktop/primary_work')

Then I just moved it to a folder I have called “programs” in my home folder. There’s a new command in my system: pwrk.

But it’s gotten me thinking. In the independent workplace of the computer, there are a lot of things that slow people down. In Linux, we deal with the cases always needed to be correct in everything. I think in my distribution I will eliminate this for system installed software. Then any issues that arise from these non-user-friendly best practices will not be from us, and we will eventually have to work out some kind of helper utility for those that get used to the easier way of doing things.

And so on.

Had a great weekend hack session. The week, the month, the year looks promising productivity-wise.
